  supplier_inquiries    Supplier quote requests
  recommended_items     Recommended products
  deals                 Timed deals (name, discount, image, category)
  reviews               Product reviews
  user_messages         Chatbot conversation history

Security: Row Level Security (RLS) enabled on all user tables
  - orders: users see own; admin sees all
  - favorites: users see own only
  - cart_items: users see own only
  - notifications: users see own only
  - discount_messages: users see own; admin can update
  - supplier_inquiries: users see own; admin can update
  - user_messages: users see own chat history only

Key Features: JSONB columns for flexible data, UUID foreign keys to auth.users,
              indexes on user_id and status columns

4. SECURITY MEASURES
================================================================================

A. Authentication & Authorization (Backend Guards):
   - auth.guard.ts          Validates Bearer token via supabase.auth.getUser()
                            -> 401 Unauthorized if missing/invalid
   - admin.guard.ts         Checks user.email === 'rehanilyas20196@gmail.com'
                            -> 403 Forbidden for non-admins
   - optional-auth.guard.ts Attaches user if token present, else anonymous

B. Supabase Auth:
   - Email/Password signup & login
   - Google OAuth sign-in
   - Password hashing (bcrypt via Supabase)

C. Cloudflare Turnstile (CAPTCHA):
   - Verified server-side on signup and login endpoints
   - Frontend renders Turnstile widget

D. HTTP Security (Helmet - Backend):
   - crossOriginResourcePolicy: cross-origin
   - contentSecurityPolicy: false (handled by frontend/Vercel)
   - xssFilter: true
   - noSniff: true
   - hidePoweredBy: true
   - frameguard: deny
   - hsts: maxAge=31536000, includeSubDomains, preload
   - referrerPolicy: strict-origin-when-cross-origin

E. CORS (Backend):
   - Allows origins from FRONTEND_URL env var (comma-separated)
   - Allows localhost and any .vercel.app domain
   - Methods: GET, POST, PATCH, DELETE
   - Headers: Content-Type, Authorization
   - Credentials: true

F. Rate Limiting (@nestjs/throttler):
   - Global:      100 requests / 60 seconds
   - Signup:       5 requests  / 60 seconds
   - Login:       10 requests  / 60 seconds
   - Returns HTTP 429 Too Many Requests

G. Input Validation (Backend):
   - Global ValidationPipe with whitelist: true, forbidNonWhitelisted: true
   - class-validator decorators (@IsEmail(), @MinLength(8), @MaxLength(), etc.)
   - File upload validation: max 5MB, image types only

H. Frontend Security (Vercel Headers in vercel.json):
   - X-Frame-Options: DENY
   - X-Content-Type-Options: nosniff
   - Referrer-Policy: strict-origin-when-cross-origin
   - Content-Security-Policy: restrictive (default-src 'self'; scripts Cloudflare)
   - Cache-Control: 1-year immutable for static assets

I. Password Handling:
   - Password pepper (PASSWORD_PEPPER env var) appended before Supabase hashing
   - Frontend validation: min 8 chars, 1 uppercase, 1 lowercase, 1 number, 1 special

5. AI CHATBOT
================================================================================

Implementation: frontend/src/components/Messages.jsx (rendered via Chatbot.jsx)

Two-Tier Architecture:

TIER 1: Google Gemini 1.5 Flash (Primary AI)
  - API Key: VITE_GEMINI_API_KEY (frontend .env, direct browser call)
  - Model: gemini-1.5-flash
  - Endpoint: https://generativelanguage.googleapis.com/v1beta/models/gemini-1.5-flash:generateContent
  - System Prompt: 110+ lines covering all store features, products, policies, FAQs
  - Generation Config: temperature 0.7, maxOutputTokens 500
  - Context: Last 10 messages sent as conversation history
  - Instructed to NEVER refuse to answer

TIER 2: Keyword-Based Fallback (When Gemini fails/unavailable)
  - 21 support topics with keyword matching (shipping, tracking, returns, refunds,
    quality, payment, cancellation, availability, bulk orders, warranty, account,
    navigation, search, cart, favorites, checkout, contact, profile, FAQ, admin, orders)
  - Product matching: searches product names, returns stock/price info
  - Order matching: detects #orderNumber patterns
  - Question bank: 20 templates x 19 topics = 380 pre-generated questions
  - Fallback answer: encourages user to ask about specific topics

Additional Chat Features:
  - Persistent chat history in user_messages table via /api/messages
  - Login-aware: syncs history when logged in; local-only otherwise
  - 5 quick-action buttons (Shipping times, Return policy, Payment methods, Track my order, Bulk order)
  - "Powered by Gemini AI" attribution

7. DEPLOYMENT
================================================================================

Deployment Config:
  - Frontend: Standard Vite build, deployed to Vercel
  - Backend: Pre-compiled with nest build (tsc) to dist/, then api/index.js

Key Deployment Detail:
  - Vercel's @vercel/node uses esbuild which lacks emitDecoratorMetadata support
  - Solution: nest build -> dist/ folder -> api/index.js loads via require()
  - Dynamic require(path.join(...)) prevents esbuild from bundling dist files
  - ConfigModule.forRoot() with ignoreEnvFile: true in production
